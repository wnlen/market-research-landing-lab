# mvp-freeze

## 用途

把开放式讨论收敛为一个主方向、一个备选方向和明确的 MVP 范围。

## 输入

- 候选方向比较结果
- 已确认的约束
- 已确认的第一收费点假设

## 输出

- 更新 `docs/08-final-decision.md`
- 更新 `docs/09-execution-roadmap.md`
- 必要时更新 `decisions/ADR-0003-final-entry-choice.md`

## 关键问题

- 第一版只做什么
- 第一版明确不做什么
- 用户第一眼看到什么
- 最小演示形态是什么
- 第一版上线形态是什么
- 后续验证什么

## 必须遵守

- 不把未验证内容包装成已确认事实
- 不继续横向扩张候选池
- 保留 `证据 / 推断 / 决策`

## 完成标准

- 主方向冻结
- 备选方向冻结
- 停止调研的条件明确
