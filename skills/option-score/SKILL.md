# option-score

## 用途

把多个候选方向放到同一评分框架下，帮助收敛选择。

## 输入

- 候选方向列表
- 已有证据与卡片
- 当前阶段约束

## 输出

- 更新 `docs/04-short-term-cashflow-options.md`
- 更新 `docs/05-long-term-moat-options.md`
- 必要时更新 `docs/07-mvp-candidates.md`

## 建议评分维度

- 短期现金流潜力
- 开发难度
- 获客难度
- 长期护城河耦合度
- 被复制风险
- 单人执行可行性
- 14 天可展示性

## 必须遵守

- 评分旁边写明依据
- 缺证据时降低置信度，不强行打满
- 结论必须保留 `证据 / 推断 / 决策`

## 完成标准

- 候选项可以横向比较
- 每个高分和低分都有依据
