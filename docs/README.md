# Docs Index

`docs/` is organized by responsibility:

- `docs/00-09`: fixed research-to-decision pipeline
- `docs/product/`: product shape, capability mapping, and first launch product design
- `docs/audits/`: repository structure and decision-chain audits
- `docs/templates/`: reusable document templates

## After Project Initialization

After `start-project` finishes, read these files in order:

1. `docs/README.md` — understand the document map.
2. `docs/00-research-question.md` — verify the initialized premises, constraints, and graduation conditions.
3. `research/raw/INDEX.md` — check whether raw materials are still in `inbox/` or have been absorbed.
4. `docs/01-market-map.md` — start the next research step.

## Entry Points

- `docs/00-research-question.md` — active research question and graduation conditions
- `docs/08-final-decision.md` — final locked direction, once available
- `docs/09-execution-roadmap.md` — execution handoff, once available
- `docs/product/product-shape.md` — long-term product shape placeholder
- `docs/product/first-launch-product-design.md` — first launch product design placeholder
